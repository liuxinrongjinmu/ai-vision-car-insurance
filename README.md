# 车险VLM智能识别系统

基于通义千问视觉语言模型(Qwen-VL)的车险业务智能化解决方案，实现车辆损伤识别、里程表读取、危险驾驶检测等多场景自动化分析。

## 项目概述

本项目是视觉大模型在车险领域的创新应用，通过多模态AI技术实现车辆相关图像的智能分析，大幅提升保险理赔和风控效率。

## 技术架构

### 核心技术栈
- **AI模型**: Qwen-VL多模态大模型 (qwen-vl-max-2025-04-08)
- **开发框架**: Python + OpenAI SDK
- **云服务**: 阿里云DashScope + OSS存储
- **数据处理**: Pandas + Excel批量处理

### 系统架构

## 核心功能

### 1. 公共配置模块 (`config.py`)

提供所有脚本共享的基础功能：
- **API客户端创建**: 从环境变量安全读取API Key
- **图片URL解析**: 支持单图/多图格式，使用`ast.literal_eval`安全解析
- **OSS URL拼接**: 自动补全图片后缀和基础URL
- **重试机制**: API调用失败自动重试（最多3次，指数退避）
- **速率控制**: API调用间隔1秒，避免触发限流

### 2. 批量图像分析 (`1-Qwen-VL-保险识别-cn.py`)

**主要功能**:
- 读取Excel中的分析模板
- 批量处理车险相关图像
- 自动生成带日期的分析报告
- 中间结果每5条自动保存，防止中断丢失
- 异常处理：单条失败不影响整体任务

**处理流程**:
1. 读取`prompt_template_cn.xlsx`中的分析任务
2. 将图片名称转换为OSS URL
3. 调用Qwen-VL模型进行多模态分析（带重试）
4. 将结果保存到`prompt_template_cn_result-{日期}.xlsx`

### 3. 单图测试与多轮对话 (`2-Qwen-VL-chat1.py`)

**功能特点**:
- `single_image_test()`: 快速单图分析测试
- `multi_turn_test()`: 多轮对话交互测试
- 实时查看分析结果

### 4. Excel驱动工作流

**输入模板结构**:
| prompt | image |
|--------|-------|
| 分析提示语 | 图片文件名或[文件名1, 文件名2] |

**输出结果结构**:
| prompt | image | response |
|--------|-------|----------|
| 分析提示语 | 图片文件名 | VLM分析结果 |

## 应用场景

### 车辆损伤识别
- **里程表读数识别**: 自动识别车辆行驶里程
- **车辆身份验证**: 通过车牌号等识别车辆信息
- **损伤评估分析**: 分析车辆剐蹭、碰撞等损伤情况

### 承保风控
- **车辆状况评估**: 评估投保车辆的实际状况
- **危险驾驶检测**: 识别潜在的危险驾驶行为
- **欺诈检测**: 通过图像分析识别可疑情况

### 理赔自动化
- **事故现场分析**: 快速分析事故现场图像
- **损失程度评估**: 自动评估车辆损失程度
- **理赔材料审核**: 自动审核理赔相关图像材料

## 使用指南

### 环境准备
```bash
# 安装依赖
pip install -r requirements.txt

# 配置环境变量（必须）
# Windows:
set DASHSCOPE_API_KEY=your-api-key
# Linux/Mac:
export DASHSCOPE_API_KEY=your-api-key
```

### 快速开始

#### 1. 批量分析
```bash
# 1. 准备分析模板（编辑prompt_template_cn.xlsx）
# 2. 上传图片到OSS存储
# 3. 运行批量分析脚本
python 1-Qwen-VL-保险识别-cn.py

# 4. 查看分析结果（文件名自动带日期）
```

#### 2. 单图测试
```bash
# 修改2-Qwen-VL-chat1.py中的图片URL和提示语
python 2-Qwen-VL-chat1.py
```

### 配置说明

**API配置**: 通过环境变量`DASHSCOPE_API_KEY`配置，不再硬编码在源码中。

**图片存储配置**:
- 图片存储在阿里云OSS: `https://vl-image.oss-cn-shanghai.aliyuncs.com/`
- 支持jpg/png格式图片
- 图片文件名在Excel中配置

**可调参数**（在`config.py`中修改）:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `DEFAULT_MODEL` | qwen-vl-max-2025-04-08 | 默认使用的模型 |
| `API_CALL_INTERVAL` | 1.0秒 | API调用间隔 |
| `MAX_RETRIES` | 3 | 最大重试次数 |
| `RETRY_WAIT` | 5秒 | 重试等待基础时间 |
